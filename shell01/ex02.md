# Comando usado para resolver:
```shell
find -type f -name "*.sh" | sed 's@.*/@@' | sed 's/\.[^.]*$//'
```
ou
```shell
find -type f -name "*.sh" -exec basename -s .sh '{}' \;
```

## Explicação dos comandos do sed:
1. ```find -type f -name "*.sh"``` serve para procurar somente arquivos (type f) que tenham um nome finalizado com .sh.
2. ```sed 's@.*/@@'``` serve para substituir tudo o que vier antes, inclusive a "/", por nada. 
    - s = substituir.
    - @ = delimitador.
    - . = corresponde a qualquer caractere, inclusive a novas linhas.
    - \* = corresponde a uma sequencia de nenhuma ou mais ocorrências dos caracteres indicados pelo ".".
    - / = caractere delimitador. Tudo o que estiver antes (.*) da "/", incluindo a "/", será substituido por nada.
    - @@ = delimitador. Dentro desta segunda parte está o que será será substituido por nada.
3. ```sed 's/\.[^.]*$//'``` serve para substituir tudo o que terminar após o ponto, incluindo o ponto, por nada.
    - s = substituir.
    - / = delimitador.
    - \\. = faz o sed interpretar o ponto como caractere e não como comando.
    - [] = uma lista para o sed.
    - ^ = representa o começo da linha.
    - [^.] = representa tudo o que não for o começo da linha. ^ dentro de uma lista, faz o sed interpretar como NOT. Neste caso tudo o que não for o começo.
    - $ = final da linha.
    - // = delimitador. Dentro desta segunda parte está o que será será substituido por nada.

## Explicação dos comandos do find:
1. ```find -type f``` = serve para achar somente ARQUIVOS.
2. ```-name "*.sh"``` = serve para procurar todos os arquivos que terminarem com .sh.
3. ```-exec``` = flag do find para executar outro comando de terminal sem escapar do find.
4. ```basename -s .sh``` = comando basename com flag para retornar o sufixo .sh.
5. ```'{}' \;``` = comando que faz parte da flag exec do find. Serve para inserir o ```basename -s .sh``` dentro do {}, executar no terminal e escapar com ```\;```, fazendo com que volte para o comando find.
