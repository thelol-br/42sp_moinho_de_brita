# Comandos usados:
```shell
echo "obase=13;ibase=5;" $FT_NBR1 + $FT_NBR2 | tr "'"'\\"?!mrdoc' '0123401234' | sed 's/3/o/' | bc | tr '0123456789ABC' 'gtaio luSnemf'
```

## Explicação dos comandos:
1. ```echo "obase=13;ibase=5;" $FT_NBR1 + $FT_NBR2``` Echo escreveu no stdout ```obase=13;ibase=5;``` e os valores das variáveis de ambiente FT_NBR1 somando a FT_NBR2.
2. ```tr "'"'\\"?!mrdoc' '0123401234'``` O tr fez a troca dos caracteres ```\"?!``` utilizando os escapes de shell, sendo 1 aspas duplas para a aspas simples, 1 aspas simples para a aspas duplas, 1 barra invertida para a barra invertida e aspas duplas para os a interrogação e exclamação, bem como as letras MRDOC. Estes caracteres foram trocados por 01234 nos primeiros 5 e 01234 nos últimos 5, pois o exercício deixa implícito que utilizaremos um input de base 5, ou seja de [0-4] membros em cada input. Ambos os inputs foram concatenados juntos para reduzir a quantidade de código utilizado.
3. ```sed 's/3/o/'``` Após a substituição com o tr, houve a necessidade de utilizar sed para fazer a correção de `obase`, pois o tr também trocou a letra O por 3 e neste caso o sed foi utilizado como uma forma de correção.
4. ```bc``` BC é uma linguagem de calculadora de precisão arbitrária. Foi utilizado sem mais informações pois o próprio pipe que carregam as informações do sed já vem com todas as informações necessárias para realizar o cálculo proposto, ou seja, temos o obase, que significa em qual base sairá o output do cálculo; temos ibase, que é o input, os 2 valores das variáveis de ambientes já convertidos para base 5 e a operação de soma, sinalizada por um `+`.
5. ```tr '0123456789ABC' 'gtaio luSnemf'``` Este tr é responsável por traduzir o resultado da soma, que está em base 13, em uma substituição por caracteres alfabéticos, gerando o output em string de `Segmentation fault`.
