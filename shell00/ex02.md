# Comandos usados:
```shell
mkdir test0 test2
truncate -s 4 test1
truncate -s 1 test3
truncate -s 2 test4
ln test3 test5
ln -s test0 test6
chmod u=rwx,g=x,o=rx test0
chmod u=rwx,g=x,o=r test1
chmod u=rx,g=,o=r test2
chmod u=r,g=,o=r test3
chmod u=rw,g=r,o=x test4
chmod u=r,g=,o=r test5
touch -t 202006012047 test0
touch -t 202006012146 test1
touch -t 202006012245 test2
touch -t 202006012344 test3
touch -t 202006012343 test4
touch -h -t 202006012344 test5
touch -h -t 202006012220 test6
tar -cf exo2.tar test0 test1 test2 test3 test4 test5 test6 --remove-files

```

## truncate
Uso o truncate para criar um arquivo vazio e a flag -s alterar o seu tamanho de cada arquivo conforme as instruções do exercício.

## ln
Comando ln foi usado para criar um link (atalho) entre os arquivos test6 e a pasta test0. Para tornar um link simbólico, foi usado a flag -s.

## chmod
Uso o chmod para alterar as permissões do arquivo.
u = user
g = group
o = other

r = read
w = write
x = execute

Caso não apresentada a permissão ou omitida totalmente, considera-se que não tem nenhuma das 3 permissões.

## touch
Uso o touch com a flag -t para alterar a data de criação do arquivo. Para a alteração da data do link simbólico, usou-se a flag -h, assim alterando a data do link e não de seu arquivo referenciado.

[[CC]YY]MMDDhhmm[.ss]
CC: First two digit of the year
YY: Last two digits of the year
MM: Month (two-digit numeric month)
DD: Day (two-digit numeric day i.e. day of month)
hh: Hour
mm: Minutes
ss: Seconds

## tar
Uso o tar para armazenar o arquivo original mantendo as datas e permissões. As flags -c e -f, podem ser concatenadas em -cf, onde eu informo o ficheiro que será criado e quais os arquivos serão armazenados nele em sua respectiva ordem. Por último, uso o --remove-files para remover os originais.
